## Project Overview

### Problem Statement
Software projects often require changelogs to track updates, bug fixes, and new features. However, maintaining changelogs manually is time-consuming and often neglected by developers.

This project automates the changelog creation process by extracting commit history from Git repositories and converting it into a structured Markdown changelog.

---

## Objectives

- Automatically extract commit history from Git repositories
- Categorize commits using conventional commit prefixes
- Generate a structured Markdown changelog file

---

## Key Features

- Automatic Git commit parsing
- Commit categorization (feat, fix, docs, chore)
- Markdown changelog generation
- Simple command-line interface
